## Overview

This document contains structured diagnostic findings from 9 verbose pipeline traces of an Edge-RAG system performing multi-hop QA. Each finding is categorized by pipeline component. The purpose is to provide precise, evidence-based descriptions of failure modes so that a downstream LLM or human analyst can identify root causes and propose targeted fixes.

**Pipeline components:** Planner (S_P) → Navigator (S_N, hop-1 + hop-2) → Verifier (S_V)

## Category C — Bridge Entity Extractor Failures

The bridge extractor (`AgenticController._extract_bridge_entities`) runs after hop-1 retrieval. It scans the filtered hop-1 chunks and returns the top-3 candidate entities to inject into the hop-2 sub-query. It uses a 3-pass heuristic: Pass 0 (location context, GPE only), Pass 1 (surname-anchor search), Pass 2 (general proper-noun fallback with proximity/type scoring). Failures here cause hop-2 to retrieve wrong articles.

## Cross-Category Summary: Most Impactful Problems

| Rank | Problem | Traces Affected | Impact |
|---|---|---|---|
| 1 | **max_docs=5 cap cutting gold chunks after reordering** | T2, T8, T9 (and partial T4) | Correct chunks retrieved and filtered but silently dropped before SLM sees them. Answer is impossible even with correct retrieval. Reordering does not reliably promote the most answer-relevant chunks. |
| 2 | **Second gold paragraph structurally absent from all retrieval** | T1, T4, T6 | The required article is either not in corpus, not graph-linked, or not reachable via any retrieval path. No pipeline fix can compensate for absent data. |
| 3 | **Planner single-hop misclassification** | T1, T2, T6 | Prevents hop-2 retrieval from running. The second gold paragraph (which contains the answer) is never queried. Caused by absence of named entities or low-confidence decomposition. |
| 4 | **Bridge entity extractor returning noise entities from non-anchor chunks** | T4, T5, T7, T8 | Wrong bridge entities injected into hop-2 query, pulling retrieval toward wrong articles. The correct bridge entity is often in chunk #1 but loses the scoring competition to entities from noise chunks that happen to score higher on proximity to query keywords. |
| 5 | **NER extracting malformed, incomplete, or misspelled entities** | T1, T2, T6, T7, T9 | Cascades into wrong graph anchors, wrong entity-mention filter tokens, and wrong bridge exclusion lists. Common patterns: DATE absorbed where PERSON expected, common English words extracted as PROPN, hyphenated role names fragmented, interrogative words merged into entity spans, misspellings propagated. |
| 6 | **Verifier claim extraction failing on short proper-name answers** | T5, T7 (and likely others) | Low confidence reported even for correct verbatim answers. 0 extracted claims means no post-hoc verification occurs. Does not cause wrong answers in these traces but reduces reliability signaling. |
| 7 | **EM evaluation penalizing correct short-form name answers** | T4 | "Reggie Jackson" vs "Reginald Martinez 'Reggie' Jackson." Factually correct answer scored as wrong by strict EM. Not a pipeline failure but an evaluation artifact that deflates reported accuracy. |
